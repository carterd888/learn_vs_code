What did i learn today?

Firstly after sharing this blog with my buddy group yesterday, i recieved feedback that the image was too big and could do with being made smaller.

i initially decided to set a height and width for the image by defining an ID and setting pixel values using CSS

this worked for the user who viewed using a web browser but it was still not displaying correctly on mobile

i reasarched responsive sizing using CSS and found this example on w3 schools

i then applied this to all images by using the element selector img

another thing i discovered today was a site used for HTML markup validation

Link to site

using this i was able to test my pages i had already created and was suprised to learn errors

the first was that i had not defined a "lang" within the html tag. this was an oversight by me in that everything on my page is in english, so i assumed the page would be read in english. however it is best practise to define this.

another error which the validator picked up is that many of my .jpg files had names containing spaces. while Visual Studio Code was able to read my sources references spaces due to the files being stored locally it is invalid syntax for a URL to contain a space

as the files were on my local machine i had 2 options, either rename the files to not contain spaces, or to encode the spaces using %20

i decided to do the former as Visual Studio Code offers shortcuts when referencing files and if i had to manually over-ride the url to encode the spaces it would defeat the purpose.
